# Feature Scope

This document defines what is included and excluded from the initial version of IssuePulse.

---

## Must-Have Features (MVP)

| Feature | Description |
|------|------------|
| User Authentication | Secure access to the platform |
| Role-Based Access | Different permissions for users |
| Issue Reporting | Core reporting functionality |
| Duplicate Detection | Prevent repeated complaints |
| Geo-Location Support | Area-based insights |
| Issue Status Tracking | Transparency |
| Civic Health Score | Area-level infrastructure metric |

---

## Good-to-Have Features (Future Scope)

| Feature | Description |
|------|------------|
| Image Uploads | Visual proof |
| SLA Breach Alerts | Accountability |
| Public Dashboards | Transparency |
| Open Data API | Research & policy use |

---

## Intentionally Excluded Features

| Feature | Reason |
|------|--------|
| Public Comment Threads | Noise and moderation complexity |
| Star Ratings | Low reliability |
