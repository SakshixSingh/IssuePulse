# Issue Lifecycle

Every issue in IssuePulse follows a defined lifecycle to maintain consistency and accountability.

---

## States

1. Reported  
   - Issue submitted by a citizen

2. Verified  
   - Issue approved by a moderator

3. In Progress  
   - Authority has started working on it

4. Resolved  
   - Issue marked as fixed

5. Closed  
   - Resolution confirmed and archived

---

## Rejected State

- If an issue is found to be false or spam, it is marked as Rejected and removed from public visibility.

---

## Purpose of Lifecycle

- Prevents ambiguity
- Enables SLA tracking
- Supports analytics and reporting
- Ensures transparency
